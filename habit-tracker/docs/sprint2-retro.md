# Sprint 2 Retrospective

## What Went Well
- Applying the Sprint 1 improvements actually worked. Committing in smaller, more frequent increments felt easier this sprint, not just something I reminded myself to do.
- Sprint 2 was noticeably easier than Sprint 1, since it was building on top of an already working foundation, the streak logic, DTOs, exception handling, and CI pipeline were all already solved, so this sprint was mostly extension, not new problem solving.
- Story 3 and Story 5 followed the same patterns already established in Sprint 1, which made planning and implementation faster and more confident.

## What Was Difficult
- Minimal new difficulty this sprint, most of the friction from Sprint 1 (CI setup, git workflow, entity design) did not repeat, since those problems were already solved and reused.

## Lessons Learned Across Both Sprints
- Planning before coding is not an extra task that slows you down, it is what makes the actual coding faster and less error prone. This became clear comparing how much smoother Sprint 2 felt after establishing good architecture and habits in Sprint 1.
- Committing small and often becomes easier with repetition, it started as something I had to consciously remember in Sprint 1, and became more natural by Sprint 2.
- If I were starting this whole assignment over from day one, I would spend more time up front on planning and architecture before writing any code, since that investment paid off directly in how much easier Sprint 2 was to build.
